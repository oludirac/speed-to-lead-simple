# Security Notes

The blueprint files have been anonymised for portfolio publishing.

## Redactions Applied

The original Make.com exports contained client-identifying and environment-specific values. These have been replaced with neutral placeholders in the checked-in scenario files.

Redacted categories include:

- Client/business names.
- Real phone numbers.
- Real email addresses.
- Live Tally and WhatsApp links.
- Make webhook IDs and datastore IDs.
- Third-party connection labels.

## Before Publishing Publicly

Before publishing, still review:

- Scenario JSON for any business-specific wording you do not want public.
- Screenshots for customer data, account IDs, run history, webhook URLs, or live links.
- Any future exported blueprints before committing them.

## Current Repo State

The scenario JSON files have been moved, renamed for clarity, and anonymised. They are suitable for documentation review, but they still require real Make.com connections and live app resources before they could be used in another Make workspace.
